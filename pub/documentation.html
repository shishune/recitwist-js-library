ReciTwist.js

Getting Started

To get started, include:

  • this ReciTwist javascript file
  • this ReciTwist css file
  • this script to call the recitwist javascript file in your html: 
<link rel="stylesheet" href="recitwist.css"> 
<script defer type="text/javascript" src='recitwist.js'></script>
  • There are no external modules needed for this library

Here is how to demonstrate some basic functionality:

First we will create an Ingredient List


const ingredientsList = new IngredientList(".IngredientsList", "Ingredients"); 

// This creates an ingredients list with a title of "Ingredients" that will go into any div with the "IngredientsList" classname. 

// Next we will add ingredients to this list 
ingredientsList.addIngredient("2 teaspoons Kosher salt")
ingredientsList.addIngredient("1 teaspoon Sugar ")
ingredientsList.addIngredient("1 pound (1-inch-thick) Bone-in pork rib chops", 
                                ["Pork shoulder steak", 
                                "Boneless, skinless chicken thighs", 
                                "Skirt or flank steak cut into 4-inch sections"])
// This ingredient has a list of alternative ingredients that are placed as the second parameneter in a list

Second we will create a set of Instructions


const instructions = new Instructions(".recipeInstructions", 
    {title: "Instructions", crossout: true,
    description:"desc: A little bit of sugar mixed into the salt helps these chops brown during their relatively short cook time, 
    and while they rest, make the wilted greens in the pork drippings."})
    // This creates a list of instructions. This instruction list has a title and description, 
    // and also allows the user to click on each step so that they can cross out each step as they go through the recipe

instructions.addStep({title: "Season meat", 
    description: "Mix kosher salt and sugar in a small bowl. Lightly pound the pork chops with a meat mallet, rolling pin, 
    or the heel of your hand until they're about 1 inch thick. Pat dry, then season all over with the salt-sugar mixture. 
    Let chops sit while you prep the radicchio and onion. ", 
    estimatedTime:3})
    // This will add a step to the instructions list. Information like title and description can be added. 
    // Each step also requires an estimated time, which will be used in the timeline. If this is not given, it defaults to 0. 
    
    
instructions.addStep({title: "Cook meat",
    description: "Press down to ensure that the center of each chop is making good contact with the hot oil and the pan. 
    Cook, turning every minute, until the chops are very well browned with some charred spots on the fattiest areas, 5 to 6 minutes total. 
    They will still be a little pink—if you like yours well-done, add another 1 or 2 minutes of cooking time. Transfer to a large plate to rest.", 
    estimatedTime:6})

Finally, we will create a timeline for those instructions

instructions.addTimeline(".timeline", {width: "1000px", height:"30px"})
// this timeline has a specified height and width, and will be added to divs with the classname "timeline"

API Documentation

Ingredients List

  IngredientList(querySelector, title = "", list = [])
  • Initialize an Ingredient List:
    • querySelector: [String] the class or id of div in your HTML that you want to add the ingredient list to
    • title: (optional) [String] title of your Ingredient list that will display at the top
    • list: (optional) [String] the list of ingredient objects that you already have made. This is not encouraged, but possible. 
IngredientList.addIngredient(ingredientName, alternatives = [], style={})
  • Add an Ingredient to your Ingredient List:
    • ingredientName: the name of the ingredient you are adding
    • alternatives: (optional) a list of alternative ingredients users may also use
    • style: (optional) object that has specific ways the developer may style the ingredient
      • style.buttonTextColour: (optional) [String]changes the button text colour. The default is “black”
      • style.buttonColour: (optional) [String] changes the button background colour. The default is “transparent”
      • style.borderRadius: (optional) [String] changes the button border radius. The default is “5px”
      • NOTE: These can also be changed in the css
IngredientList.changeViewAlternativesButtonText(newText)
  • [String] Changes the text of the button that allows users to view the alternative ingredients to newText. Default is “View Alternatives”. 
IngredientList.changeHideAlternativesButtonText(newText)
  • [String] Changes the text of the button that allows users to hide the alternative ingredients to newText. Default is “Hide Alternatives”. 
IngredientList.changeSwapButtonText(newText)
  • [String] Changes the text of the button that allows users to swap the alternative ingredients when the alternative view is expanded to newText. Default is “Swap”. 
IngredientList.changeModalSwapText(newText)
  • [String] Changes the title text inside the alternative ingredients modal to newText. Default is “Swap”.

Ingredient

Ingredient(name, alternatives = [], text, style)
  • Initialize an ingredient
    • name: [String] name of the ingredient
    • alternatives: (optional) [list[String]] the names of alternative ingredients
    • text: [Object] of the texts that will appear in the DOM Element representation of an ingredient. Includes
      • text.viewAlternativesText: [String] the text of the button that allows users to view the alternative ingredients.
      • text.hideAlternativesText: [String] the text of the button that allows users to hide the alternative ingredients.
      • text.swapText: [String] the text of the button that allows users to swap the alternative ingredients when the alternative view is expanded.
      • text.modalSwapText: [String] the title text inside the alternative ingredients modal. 
Ingredient.createElement()
  • Will return a DOM element that includes all of the Ingredient information, along with the alternative ingredients and necessary functions.
    • Will create a modal for the alternative ingredients
    • Will create a div for the alternative ingredients
    • Will create a button to view/hide the alternative ingredients
    • Will create buttons for each alternative ingredient so that they can be swapped with the original ingredient

Instructions

Instructions(querySelector, instructions)
  • Initializes instructions
    • querySelector: [String] the class or id of div in your HTML that you want to add the instructions to
    • instructions: object of instruction information. All of these fields are optional
      • instructions.title: [String] title that will appear before the list of instructions
      • instructions.description: [String] description that will appear after the title
      • instructions.crossout: [boolean] if true allows the user to click on each step to cross it out. Note: if used with the timeline, it will also turn the corresponding time block in the timeline into a different colour!
      • instructions.stepName: (optional) [String] The word displayed to denote each step. Default is “Step” 
Instructions.addStep(stepInfo)
  • Adds a step to the instructions
    • stepInfo: [Object] that includes step information. All of these fields are optional
      • Please view the Step Documentation for further details 
Instructions.addTimeline(querySelector, style)
  • Creates and adds a timeline to the DOM.
  • NOTE: This should be called AFTER all of the steps have been added.
    • querySelector: [String] the class or id of div in your HTML that you want to add the timeline to
    • style: [Object] of the style of the timeline. Please view the Timeline Documentation for further details

Step

Step(index, id, step, crossout)
  • Initializes a step
    • index: [Int] index of the step in relation to the rest of the other steps.
    • id: [String] the id used to refer to the step. This is used in the timeline to link to this step. Typically will follow the format [instruction querySelector]-[step index]
    • step: [Object] of step information
      • step.stepName: [String] What a step will be referred to. Default is “Step”
      • step.seperator: [String] What is put between the Step name and the title of the step. Default is " "
      • step.title: [String] The title of the step. Default is ""
      • step.description: [String] The description of the step. Default is ""
      • step.estimatedTime: [float] The estimated time it takes to complete the step. Default is 0
    • crossout: [boolean] if true allows the user to click on each step to cross it out. Note: if used with the timeline, it will also turn the corresponding time block in the timeline into a different colour! 
Step.createElement()
  • Will return a DOM element that includes all of the Step information. All the information is wrapped in a section element so that it can be linked to. Each Step Element can be clicked to become crossed out 
Step.addTimeline(timeline)
  • Will add a timeline to this step.
    • timeline: [Timeline] this links the Step to the timeline so that when it is marked as complete, the timeline will also reflect those changes. 
Step.setStepAsComplete()
  • Will set the step as complete and also call the timeline to set the corresponding time block as complete

Timeline

Timeline(querySelector, steps, style={})
  • Initializes a timeline. Note that the blocks within the timeline will be refered to as “time blocks”
    • querySelector: [String] the class or id of div in your HTML that you want to add the instructions to
    • steps: [List [Step]] Steps to base the timeline off of
    • style: (optioal) Style applied to the timeline
      • style.padding:[String] Padding applied to the div around the time blocks (ie the timeline padding). Default is “0px 2px”.
      • style.bgColour: [String] Background colour of the div surrounding the time blocks (ie the timeline background colour). Default is "white"
      • style.borderRadius:[String] Border radius of the timeline. Default is “5px”
      • style.height:[String] Height of each time block inside the timeline. Default is “30px”
      • style.width:[String] Wdith of each time block inside the timeline. Default is “1000px”
      • style.maxModalWidth:[String] The maximum width of the modal for each time block. Default is “250px”
      • style.maxModalHeight:[String] The maximum height of the modal for each time block. Default is “500px”
      • style.inprogressColour:[String] The colour of time blocks that are in progress. Default is “red”
      • style.completedColour:[String] The colour of time blocks that are complete. Default is “green”
      • style.border:[String] The border style of the time blocks. Default is “none” 
Timeline.createTimeline()
  • Will create and return a DOM element with all of the timeline information 
Timeline.addTimeline()
  • Will create and add a timeline to the corresponding DOM element according to querySelector 
Timeline.setBlockAsComplete(index)
  • Will set the index-th time block as complete (or uncomplete if already complete)
